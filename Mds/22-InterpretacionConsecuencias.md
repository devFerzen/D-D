# Resultado de acciones y consecuencias

Base canonica: `00-PuntosFijosCanonicos.md`.
Este archivo desarrolla PF-02, PF-03 y PF-10.
No redefine decision base (ver `01-MecanicaBaseResolucion.md`).

## Interpretacion post-tirada

- Exito claro:
  objetivo cumplido sin coste relevante.

- Exito parcial:
  objetivo cumplido con coste, retraso o nueva tension.

- Fallo con consecuencia:
  no se cumple el objetivo y la escena cambia.

## Regla de progresion
Un fallo no debe congelar la partida.
Debe generar nueva situacion jugable.

## Formato minimo del DM (PF-10)
1. Escena
2. Resultado
3. Estado actualizado
4. Tiradas requeridas (si aplica)
